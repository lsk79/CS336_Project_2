  Input: "In a Python script running on a Linux server (ILab), how do I connect to a PostgreSQL database using psycopg2, prompt for credentials using getpass, execute a SELECT query provided as part of the command line argument, and format the output results for display?"
  Output: "Here’s a clean way to do exactly what you’re describing on a Linux server (like iLab): connect to PostgreSQL with psycopg2, securely prompt for credentials using getpass, take a SQL SELECT query from the command line, execute it, and print formatted results.

import psycopg2
import sys
from getpass import getpass

def main():
    if len(sys.argv) < 2:
        print("Usage: python script.py \"SELECT * FROM table;\"")
        sys.exit(1)

    query = sys.argv[1]

    # Secure credential input
    db_name = input("Database name: ")
    user = input("Username: ")
    password = getpass("Password: ")

    try:
        conn = psycopg2.connect(
            dbname=db_name,
            user=user,
            password=password,
        )

        cur = conn.cursor()
        cur.execute(query)
        conn.commit()
        cur.close()
        conn.close()

    except psycopg2.Error as e:
        print("Database error:", e)

if __name__ == "__main__":
    main()

Here are the standard approaches (use these in order of preference):

Option 1: Use a virtual environment (BEST / recommended)

This is what professors usually expect.
1. Create a virtual environment
python3 -m venv myenv
2. Activate it
source myenv/bin/activate

Your terminal should now show:
1. Create a virtual environment
python3 -m venv myenv
2. Activate it
source myenv/bin/activate
Your terminal should now show:
(myenv) user@ilab:~
3. Install packages normally
pip install psycopg2-binary
4. Run your script
python script.py "SELECT * FROM table;"
5. Exit when done
deactivate

Option 2: Install with --user (no virtual environment)
If venv is annoying or not allowed:
pip install --user psycopg2-binary
This installs into:
~/.local/lib/pythonX.X/site-packages
Make sure Python sees it (usually automatic on iLab, but if not):
export PYTHONPATH=$HOME/.local/lib/python3.X/site-packages:$PYTHONPATH
